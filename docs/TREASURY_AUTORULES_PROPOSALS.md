## Предложения по автоправилам
Ниже — конкретные правила (match_type = `contains`, если не указано иначе).  
Указаны ориентировочные статьи и флаг `create_dds` (настройка на вашей стороне).

### 1) Депозиты (универсальное правило вместо 2 точечных)
- **Матч:** `purpose contains "депозит"`  
- **Если amount < 0:** действие `ignore` (размещение депозита)  
- **Если amount > 0 и purpose contains "возврат":** `ignore`  
- **Если purpose contains "процент"** → `categorize` → статья «Прочие доходы / Проценты по депозиту», `create_dds=true`

### 2) Беспроцентный займ от учредителя
- **Матч:** `purpose contains "беспроцентный займ от учредителя"`  
- **Действие:** `categorize` → статья «Займы / Учредитель»  
- **create_dds:** обычно `false` (если не хотите в ДДС)

### 3) Аренда (аванс)

## Предложения по новым связям (relationships)
### 1) Транзакция ↔ Контрагент
**Поле:** `company_id` (FK на `companies`)  
**Правило:** авто‑сопоставление по `payer_inn` / `payee_inn`.  
**Плюс:** фильтры по контрагентам, аналитика оборотов.

### 2) Транзакция ↔ Договор/Сделка/Проект
**Поля:** `contract_id`, `deal_id`, `project_id`  
**Правило:** извлекать номер договора/сделки из `purpose` (regex) + ручное подтверждение.  
**Плюс:** кассовые потоки по проектам.

### 3) Транзакция ↔ Документ (счет/акт/платежка)
**Поле:** `document_id`  
**Правило:** связка по `doc_num + дата + сумма`, ручная привязка в UI.  
**Плюс:** закрытие цепочки «счет → платеж → факт».

### 4) Транзакция ↔ Статья ДДС
**Поле:** `income_expense_id` или `dds_id`  
**Правило:** заполнение по `category_code` или срабатыванию автоправила.  
**Плюс:** автогенерация ДДС и снижение ручных операций.

### 5) Транзакция ↔ Банковский счет
**Поле:** `bank_account_id`  
**Правило:** сопоставление по счету из выписки.  
**Плюс:** контроль остатков и сверка по счетам.

### 6) Транзакция ↔ Согласование
**Поле:** `approval_task_id`  
**Правило:** если `amount > X` или `контрагент в рисковых` — создавать задачу согласования.
